Regence Style Guide

Content Styles

Content

This basic container is used to wrap a variety of content areas and plays an important role in setting correct margins and padding. By itself it only adds a white background.

Example page header Subtext for header

Any content can be added to this container.

<div class="content">
  <div class="page-header">
    <h1>Example page header <small>Subtext for header</small></h1>
  </div>
  <p>Any content can be added to this container.</p>
</div>

Landing Content

The content on landing pages has a white background without any default padding. The content in these section is completely custom and images and carousels are flush against the edges. Padding is instead applied to the .page-header. Other than the page header, only specialized content can be added to this container.

Example page header Subtext for header

Specialized content, mainly hero units, are added to this container.

<div class="content landing-content">
  <div class="page-header">
    <h1>Example page header <small>Subtext for header</small></h1>
  </div>
  <div class="hero-unit hero-inverted hero-image hero-shopping">
    <div class="image image-responsive" style="background-image: url(/assets/img/landings/shopper/masthead.jpg)"></div>
    <div class="carousel-caption">
      <p class="burgundy">Specialized content, mainly hero units, are added to this container.</p>
    </div>
  </div>
</div>

Main Content

Our main content div is used on most articles and has padding to improve legibility. This style is dependent on whether the article has a left or right sidebar, adding negative margins and borders depending on the parent class.

Example page header Subtext for header

Any content can be added to this container.

<div class="content main-content">
  <div class="page-header">
    <h1>Example page header <small>Subtext for header</small></h1>
  </div>
  <p>Any content can be added to this container.</p>
</div>

Sidebar Content

The sidebar content div is meant to be used in the right rail. It has a clear background and less padding than the main content.

Regence Corporate

Portland OR 97217

Forms

Default form

Individual form controls receive styling, but without any required base class on the <form>or large changes in markup. Results in stacked, left-aligned labels on top of form controls.

Legend Example block-level help text here. 
<form>
  <legend>Legend</legend>
  <label>Label name</label>
  <input type="text" placeholder="Type something…">
  <span class="help-block">Example block-level help text here.</span>
  <label class="checkbox">
    <input type="checkbox"> Check me out
  </label>
  <button type="submit" class="btn">Submit</button>
</form>

Horizontal form

Right align labels and float them to the left to make them appear on the same line as controls. Requires the most markup changes from a default form:

  • Add .form-horizontal to the form
  • Wrap labels and controls in .control-group
  • Add .control-label to the label
  • Wrap any associated controls in .controls for proper alignment
Legend 
<form class="form-horizontal">
  <legend>Legend</legend>
  <div class="control-group">
    <label class="control-label" for="inputEmail">Email</label>
    <div class="controls">
      <input type="text" id="inputEmail" placeholder="Email">
    </div>
  </div>
  <div class="control-group">
    <label class="control-label" for="inputPassword">Password</label>
    <div class="controls">
      <input type="password" id="inputPassword" placeholder="Password">
    </div>
  </div>
  <div class="control-group">

Checkboxes and radios

Checkboxes are for selecting one or several options in a list while radios are for selecting one option from many.

Default (stacked)
