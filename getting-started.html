Getting started

Overview of the project, its contents, and how to get started with a simple Liferay theme.

1. Download

Before downloading, be sure to have Liferay Developer Studio, or a similar IDE, and some working knowledge of HTML and CSS.

Get the theme, layouts, plugins, and sample content. This is the easiest way to get started.

Download zip

2. File structure

Within the download you'll find the following file structure and contents, logically grouping common assets.

Once downloaded, unzip the compressed folder to see the structure of the Liferay theme. You'll see something like this:

  docroot/
  ├── WEB-INF/
  │   ├── liferay-look-and-feel.xml
  │   ├── liferay-plugin-package.properties/
  ├── _diffs/
  │   ├── [duplicate of Custom theme files]
  ├── _less/
  │   ├── [Custom preprocessed CSS files]
  └── color_schemes/
  │   ├── [Custom alternate CSS theme files]
  └── css/
  │   ├── [Liferay and Custom styles]
  ├── documents/
  │   ├── [Custom theme support files]
  ├── images/
  │   ├── [Liferay dependent images]
  └── js/
  │   ├── [Liferay and Custom javascript files]
  └── templates/
      ├── [Liferay and Custom template files]

This is the most basic form of Bootstrap: compiled files for quick drop-in usage in nearly any web project. We provide compiled CSS and JS (bootstrap.*), as well as compiled and minified CSS and JS (bootstrap.min.*). The image files are compressed using ImageOptim, a Mac app for compressing PNGs.

Please note that all JavaScript plugins require jQuery to be included.

3. What's included

This Liferay theme includes Twitter Bootstrap. The framework can be summarized with a handful of categories visible at the top of the page.

Similar to Components, these JavaScript plugins are interactive components for things like tooltips, popovers, modals, and more.

1. How to create a Liferay theme

With a brief intro into the contents out of the way, we can focus on how to integrate with Liferay. To do that, we'll have to create a Liferay theme.

The process for creating a new theme is nearly identical to the one for making a portlet. You will need both a project name (without spaces) and a display name (which can have spaces). For example, the project name could be deep-blue, and the theme title Deep Blue. We will show two ways to accomplish this. First, we will demonstrate by using Liferay Developer Studio.

Using Developer Studio: Go to File → New → Liferay Project

Next, go through the following steps to setup your new theme:

  1. Fill in regence for the Project name and Regence for the Display name
  2. Select the Liferay Plugins SDK and Portal Runtime you’ve configured
  3. Select Theme for your Plugin type
  4. Click Finish

View the Liferay documentation for other methods and options for creating a theme.

Deploying the theme

To deploy in Developer Studio: Click and drag your theme project onto your server.

Upon deploying your plugin, your server will output messages indicating your plugin is read, registered, and now available for use.

Reading plugin package for regence-theme
Registering themes for regence-theme
1 theme for regence-theme is available for use

You should get a BUILD SUCCESSFUL message, which means that your theme is now being deployed. If you switch to the terminal window running Liferay, and wait for a few seconds, you should see the message 1 theme for regence-theme is available for use.

Add custom theme files

To add files in Developer Studio: Click and drag the custom theme files onto the docroot folder in your theme.

Leave the dialog at default, Copy files and folders, and press OK.

After copying the files you'll need to redeploy your theme.

Go to your web browser and login to the portal. Then hover over Manage at the top of the page, and click on Page. Directly underneath the words Manage Pages select the Look and Feel tab. Simply click on your theme to activate it.

2. How to install Liferay layouts

With the Bootstrapped based Regence theme installed we'll now need some Bootstrap based layouts. The next steps will show you how to install and activate these layouts.

Using Developer Studio: Go to File → Import → Web → WAR file

Select browse and locate the four layouts from the custom theme download. They're located at ~/docroot/documents/layouts.

After pressing the Finish button deploy the newly installed layout the same way as the theme. Once deployed you will be able to select this layout from your web browser. Hover over Manage at the top of the page, and click on Page Layout. If you installed article-left-sidebar, then you can select Article Left Sidebar and press Save to activate it.
